# 桌宠开发计划

## 1. 项目结构

```
debian-app/
├── websocket_api.md          # API 文档（已存在）
├── desktop_pet/              # 桌宠应用主目录
│   ├── __init__.py
│   ├── main.py               # 应用入口
│   ├── pet.py                # 桌宠核心类（动画、状态管理）
│   ├── websocket_client.py   # WebSocket 客户端
│   ├── ui.py                 # UI 界面管理
│   ├── config.py             # 配置管理
│   └── assets/               # 资源目录
│       ├── idle/              # 待机动画帧
│       ├── talking/           # 说话动画帧
│       ├── happy/             # 开心动画帧
│       ├── sad/               # 难过动画帧
│       └── thinking/          # 思考动画帧
├── requirements.txt          # 依赖列表
└── README.md
```

## 2. 核心模块设计

### 2.1 配置管理 (config.py)
- WebSocket 连接地址（默认 `ws://127.0.0.1:8765`）
- 认证 token（可选）
- 动画帧率设置
- 窗口属性设置

### 2.2 WebSocket 客户端 (websocket_client.py)
- 连接到 WebSocket 服务器
- 发送消息（JSON 格式）
- 接收流式响应（解析 SSE 格式）
- 错误处理和重连机制
- 回调机制通知 UI 更新

### 2.3 桌宠核心类 (pet.py)
- **动画状态机**：idle, talking, happy, sad, thinking
- **PNG立绘展示**：每种状态对应不同的 PNG 文件
- **状态切换逻辑**：根据 WebSocket 消息自动切换动画状态

### 2.4 UI 界面管理 (ui.py)
- **Pygame 窗口初始化**：无边框、透明背景
- **消息输入框**：左侧文本输入
- **角色png展示**：在窗口右侧展示当前状态的 PNG 图像
- **聊天记录显示**：不显示历史消息只在左侧显示当前消息
- **系统托盘**：使用 pystray 库实现

### 2.5 应用入口 (main.py)
- 初始化所有模块
- 主事件循环
- 系统托盘菜单（显示/隐藏、退出）
- 窗口拖拽功能

## 3. 动画实现方案

### 3.1 帧序列命名规范
```
idle/
  ├── idle_001.png
  ├── idle_002.png
  └── ...

talking/
  ├── talking_001.png
  ├── talking_002.png
  └── ...
```

### 3.2 状态切换规则
| 事件 | 切换到状态 | 说明 |
|------|-----------|------|
| 发送消息 | thinking | 用户输入中 |
| 收到消息 | talking | AI 回复时 |
| 回复完成 | happy | 正常回复完成 |
| 收到错误 | sad | 发生错误 |
| 空闲时 | idle | 无交互 |

### 3.3 帧率配置
- 待机动画：8 FPS
- 说话动画：12 FPS
- 其他动画：10 FPS

## 4. WebSocket 通信流程

### 4.1 发送消息
```python
{
    "text": "用户输入的文本",
    "session_id": "固定会话ID"
}
```

### 4.2 接收响应
- 解析 SSE 格式：`data: {"type": "text", "content": "..."}\n\n`
- `type: text` → 更新消息内容
- `type: finish` → 结束当前回复
- `type: error` → 显示错误状态

## 5. 系统托盘功能

### 5.1 托盘菜单
- 显示窗口
- 隐藏窗口
- 退出程序

### 5.2 实现方式
使用 `pystray` 库创建系统托盘图标，配合 `Pillow` 处理图像。

## 7. 实现步骤

### 第一步：项目初始化
- 创建目录结构
- 创建 requirements.txt
- 创建空资源目录（用户需要放入 PNG 文件）

### 第二步：配置模块
- 创建 config.py
- 定义配置类和默认值

### 第三步：WebSocket 客户端
- 实现 websocket_client.py
- 处理连接、发送、接收
- SSE 解析器

### 第四步：桌宠核心
- 创建 pet.py
- 实现动画状态机
- 实现帧序列播放

### 第五步：UI 界面
- 实现 ui.py
- 窗口初始化
- 消息输入/显示
- 透明度设置

### 第六步：系统托盘
- 集成 pystray
- 托盘菜单

### 第七步：主程序
- 整合所有模块
- 主事件循环

## 8. 注意事项

1. 窗口需要设置为 ` pygame.NOFRAME` 并使用 `set_alpha(None)` 实现透明背景
2. PNG 文件需要使用 RGBA 模式以支持透明度
3. WebSocket 连接失败时需要优雅降级
4. 系统托盘需要后台线程支持
5. 动画帧需要用户自行准备并放入对应目录
